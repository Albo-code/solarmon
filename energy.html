energy

Usage

usage: energy.py [-h] [-t QUARTER_OF_AN_HOUR] [-s site_api_key.yaml]
                 start_date end_date

Call the SolarEdge Monitoring API `energy` command to get energy generation
data and store in `.json` file.

positional arguments:
  start_date            Start date, e.g. '2021-08-01'
  end_date              End date, e.g. '2021-08-31'

options:
  -h, --help            show this help message and exit
  -t QUARTER_OF_AN_HOUR, --time-unit QUARTER_OF_AN_HOUR
                        Time unit, one of 'QUARTER_OF_AN_HOUR', 'HOUR', 'DAY',
                        'WEEK', 'MONTH', or 'YEAR'
  -s site_api_key.yaml, --site-data site_api_key.yaml
                        Name of .yaml file containing site data

Get energy data from the SolarEdge server using Python requests call to send the SolarEdge API energy command.

Parameters

  • site_id (str) – Value of site id to be used in server request.

  • api_key (str) – Value of api key to be used in server request.

  • start_date (str) – Value of energy command’s start date.

  • end_date (str) – Value of energy command’s end date.

  • time_unit (str) – Value of energy command’s time unit.
